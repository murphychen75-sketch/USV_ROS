# 终端输出降噪 — 实施记录

**定位**：记录 `usv_sim_full` 已落地的**默认安静启动**行为；与 [`QUICK_START.md`](./QUICK_START.md) 中的启动命令一致。若需恢复以往「满屏 INFO」便于排障，见下文 **`verbose_launch`**。

---

## 1. 默认行为（`verbose_launch:=false`，省略时同默认）

| 项 | 实施方式 |
|----|-----------|
| **ROS 2 节点** | 在 `main.launch.py` 非 verbose 时前置 `SetEnvironmentVariable(RCUTILS_LOGGING_SEVERITY=WARN)`；各 `Node` 默认 `output='log'` 并附加 `--ros-args --log-level warn`（由 `quiet_ros_node_kwargs()` 统一生成，见 `usv_sim_full/launch_config_helpers.py`）。 |
| **`ros_gz_bridge` / `parameter_bridge`** | `robot_bringup`、`infra_sim` 中桥接节点使用上述 `output='log'` + WARN，避免每条 bridge 一行刷终端。 |
| **`robot_state_publisher`、`ros_gz_sim/create`、`odom_tf_broadcaster`、`radar_gz_bridge`、`robot_localization`、`obstacle_spawner`** | 均在 `components/robot_bringup.launch.py` 中走同一套 quiet 规则。 |
| **RViz** | `components/visualization.launch.py` 与 `main` 内包含关系下，默认 `output='log'` + WARN。 |
| **`session_manager` 子进程** | 默认**不**传 `--verbose`；stdout 仅保留末尾 JSON。`main.launch.py` / `test_hull.launch.py` / `sensor_tune.launch.py` 在 quiet 下只打印一行摘要（船数量、`session_path`），不再 dump 整段 stdout。 |
| **`session_manager` 内 print** | `create_session(..., verbose=False)` 时跳过「Generated URDF…」「Post-processed URDF…」；`--verbose` 时恢复。 |
| **`infra_sim` 内 print** | `GZ_SIM_RESOURCE_PATH` / `GAZEBO_MODEL_PATH` / `GZ_SIM_SYSTEM_PLUGIN_PATH` 及插件异常说明仅在 **`verbose_launch:=true`** 时由 `OpaqueFunction` 打印；环境变量仍始终通过 `SetEnvironmentVariable` 注入。 |
| **真值 / 毫米波 / `scenario_manager` / `usv_sim_wrapper` / `usv_env_dynamics` / 静态 map→odom TF** | `main.launch.py` 内对应 `Node` 均使用 `quiet_ros_node_kwargs`。 |
| **Nav2 全线 + `nav2_sim_full_bringup`** | `main` 已设 `RCUTILS_LOGGING_SEVERITY=WARN`；`nav2_thruster_bringup.launch.py` 在启动 `navigation_launch` 前若为非 verbose 再次前置同一环境变量（便于单独起 Nav2 时仍降噪）；`tf_namespace_relay` 与 `cmd_vel_to_thruster` 的 `ExecuteProcess` 在非 verbose 时 `output='log'`。 |
| **`test_hull.launch.py`** | `gz sim` 非 verbose 时 `output='log'`；其余同 `main` 摘要 session、`robot_bringup` 传 `verbose_launch`、`RCUTILS` WARN。 |
| **`sensor_tune.launch.py`** | 三节点 + 可选 `RCUTILS` WARN；`session_manager` 支持 `--verbose` 联动。 |

**日志文件**：各节点写入 `ros2 launch` 启动时提示的目录（通常为 **`~/.ros/log/<日期>-<主机>-<pid>/`** 下按节点名分的日志）。

**未覆盖（预期仍可能出现在终端）**：`gz sim` 的 C++ Utils 警告、`libEGL`、部分非 rclcpp 输出；**`gy_radar_driver` 的 `radar_controller.launch.py`** 未改包内节点 `output`（依赖全局 `RCUTILS_LOGGING_SEVERITY=WARN` 压低部分 INFO）。

---

## 2. 恢复详细输出

任意入口在命令行追加：

```bash
verbose_launch:=true
```

示例：

```bash
ros2 launch usv_sim_full main.launch.py verbose_launch:=true
ros2 launch usv_sim_full nav2_sim_full_bringup.launch.py verbose_launch:=true
```

效果：`session_manager` 加 `--verbose`；infra 路径 print 恢复；各节点 `output='screen'` 且不再强制 `--log-level warn`；`main` / `test_hull` 再次打印完整 session stdout。

## 4. 说明

降噪**不消除** QoS/TF 等实质问题；若仅见 WARN/ERROR 消失而功能异常，请用 `verbose_launch:=true` 或查看 `~/.ros/log` 定位。
