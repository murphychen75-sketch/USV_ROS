# usv_simulation 仓库结构说明（docs_v3）

本文档描述 `src/usv_simulation` 下**主要目录与 ROS 2 包**的职责，便于快速定位仿真、传感器与 VRX 相关代码。版本代号 **docs_v3** 表示与早期 `docs_v1`、`docs_v2` 并列存放，后续可在根目录 README 中链接至此。

---

## 顶层一览

```
usv_simulation/
├── docs/                    # 文档（含本目录 docs_v3）
├── env_panel/               # RViz2 插件：海况/环境参数面板
├── sensor_plugins/          # 雷达与 Gazebo 插件、桥接等
├── sim_test/                # 仿真相关测试与工具（Python）
├── usv_sim_full/            # 主仿真入口：YAML 驱动、launch、脚本与资源
└── vrx/                     # Virtual RobotX（VRX）竞赛仿真资源与适配
```

---

## `usv_sim_full/`（核心启动包）

**本包内目录细表（简版）**：见 [← `usv_sim_full/仓库结构说明.md`](../../usv_sim_full/docs/仓库结构说明.md)。

| 内容 | 作用 |
|------|------|
| **角色** | 功能完备的 USV 仿真**编排中心**：依赖 Gazebo（gz）与 `ros_gz_*`，用 **YAML** 配置场景与参数。 |
| **典型内容** | `launch/`（如 `main.launch.py`、Nav2 相关 bringup）、`config/`、`scripts/`（含船体/环境动力学等）、模型与世界文件等资源路径。 |
| **依赖关系** | 与 `usv_interfaces`、`wamv_gazebo`、`usv_mmwave_sim` 等配合，构成「从世界生成到话题输出」的主链路。 |

**入门建议**：从 `launch/` 中主入口文件开始读，对照 `config` 理解一条完整仿真流程。

## `vrx/`（VRX 仿真栈）

VRX（Virtual RobotX）提供 WAM-V 水面平台、海洋环境与 Gazebo 侧资源；本目录将相关包**集中放置**，供 `usv_sim_full` 等引用。

| 包 / 子目录 | 作用 |
|-------------|------|
| **`vrx_ros`** | VRX 的 ROS 2 侧资源（节点、桥接与接口等），连接仿真与标准 ROS 话题/TF。 |
| **`vrx_gz`** | 面向 **Gazebo Sim（gz）** 的资源与 launch（如生成实体、环境、竞赛场景），依赖 `vrx_ros`、`ros_gz_sim`、`wamv_gazebo` 等。 |
| **`vrx_urdf/`** | 经典 VRX **URDF / Gazebo Classic 时代**的资产与示例，按包拆分： |

其中 `vrx_urdf/` 内常见包：

| 包名 | 作用 |
|------|------|
| **`wamv_description`** | WAM-V **机器人描述（URDF/xacro）**，几何与关节等静态模型。 |
| **`wamv_gazebo`** | 将 WAM-V **接入 Gazebo 的模板与示例**（动力学、传感器挂载等扩展入口）。 |
| **`vrx_gazebo`** | RobotX 竞赛在 Gazebo 中的**模型与资源**（偏「资源包」定位）。 |

**说明**：当前主仿真线以 **gz + ros_gz** 为主时，优先关注 `vrx_gz` + `vrx_ros`；`vrx_urdf/*` 多用于模型来源与兼容旧流程。

## `sensor_plugins/`（传感器与 Gazebo 插件）

**本目录子包与数据流（简版）**：见 [← `sensor_plugins/仓库结构说明.md`](../../sensor_plugins/仓库结构说明.md)。

| 包名 | 作用 |
|------|------|
| **`gz_maritime_radar_plugin`** | **海事雷达**的 Gazebo **系统插件**（CMake 导出），在仿真里产生雷达相关数据通路。 |
| **`radar_gz_bridge`** | 将 Gazebo（gz-transport）侧的雷达数据 **桥接到 ROS 2**，使用 `marine_sensor_msgs` 等消息。 |
| **`gy_radar_driver`**（目录名 `gy_radar_driver-main`） | **实机 / NaviRadar 类**雷达驱动（GY SDK），发布雷达与网格等话题；与仿真插件并列，便于同一接口体系下联调。 |
| **`usv_mmwave_sim`** | **毫米波仿真一体化包**：Gazebo 插件（`libusv_4d_radar_plugin.so`，注册名 `usv_4d_radar_gz::FourDRadarPlugin`）+ ROS 2 节点 `mmwave_4d_cloud_node`（gpu_ray 桥接后增强点云）；`usv_sim_full` 的 `package.xml` 依赖本包；点云话题规则见 [`docs_v2/QUICK_START.md`](../docs_v2/QUICK_START.md)。 |

**关系简述**：`gz_maritime_radar_plugin` 在仿真内产生数据 → `radar_gz_bridge` 转成 ROS 话题；毫米波链路由 **`usv_mmwave_sim`**（插件 + 点云节点）支撑；`gy_radar_driver` 偏硬件/实船。**当前仓库内 `sensor_plugins/` 含海事雷达插件、桥接、毫米波一体化包与 GY 驱动等**，详见该目录下的结构说明。

## `env_panel/`（环境与海况可视化控制）

| 内容 | 作用 |
|------|------|
| **角色** | **RViz2 面板插件**：在界面中调节 VRX 环境中的 **风、浪、流** 等，并带安全渐变（ramping），避免参数突变。 |
| **依赖** | `rviz_common`、`ros_gz_interfaces` 等，与 gz 侧环境服务/话题协同。 |

---

## `sim_test/`（测试与实验工具）

| 内容 | 作用 |
|------|------|
| **角色** | **Python** 包（`ament_python`），用于仿真相关的测试、调试或小工具（依赖含 `PyQt5`、`yaml` 等）。 |
| **使用场景** | 快速验证话题、人机小界面或实验脚本；具体入口见包内 `setup.py` 与源码。 |

---

## `docs/`（文档）

| 子目录 | 作用 |
|--------|------|
| **`docs_v1/`** | 早期使用说明、Launch、RViz、传感器扩展与排错等。 |
| **`docs_v2/`** | 增量文档（如快速开始类）。 |
| **`docs_v3/`** | **本文件所在**：以**仓库结构、模块职责**为主的说明，便于架构级查阅。 |

---

## 与仓库其他部分的衔接（只读边界）

- **`usv_interfaces`**（工作区其他包）：仿真侧发布的自定义话题/消息应尽量与其中 **topic 常量与消息定义**一致。  
- **`src/third_party/`**（工作区根下）：如 `marine_sensor_msgs` 等海事消息定义；雷达桥接与驱动包会依赖这些接口。

---

## 维护提示

- 新增 **launch / 世界 / 传感器**时，优先在 `usv_sim_full` 或对应 `sensor_plugins` 包内保持单一职责，并在 `docs_v3` 或发行说明中补一行「从属关系」。  
- 修改 **VRX 相关 launch** 时，同时核对 `vrx_gz` 与 `usv_sim_full` 的命名空间、话题是否一致。
