# sensor_plugins — 目录结构说明（简版）

本目录 **不是** 单个 ROS 包，而是**多个并列的传感器相关包/工程**的根目录。在 `usv_simulation` 中的定位见总览（双向引用）：

**→ [../docs/docs_v3/仿真仓库结构说明.md](../docs/docs_v3/仿真仓库结构说明.md)**（章节「sensor_plugins / 传感器与 Gazebo 插件」）

各子目录若自带 `README.md`（如 `radar_gz_bridge`、`gz_maritime_radar_plugin`），细节以子包为准。

---

## 当前子包一览

| 子目录 | ROS 包名 | 作用（摘要） |
|--------|-----------|----------------|
| `gz_maritime_radar_plugin/` | `gz_maritime_radar_plugin` | Gazebo（gz）侧 **海事雷达仿真插件**（CMake 包）；含模型与插件源码。 |
| `radar_gz_bridge/` | `radar_gz_bridge` | 将 gz-transport 雷达数据 **桥接到 ROS 2**（`marine_sensor_msgs` 等）；含 launch 与参数。 |
| `gy_radar_driver-main/` | `gy_radar_driver` | **NaviRadar / GY SDK** 实机雷达驱动与相关节点（控制、转换、网格、NMEA 等）。 |
| `usv_mmwave_sim/` | `usv_mmwave_sim` | **毫米波仿真一体化**：Gazebo 系统插件（4D/验证）+ `mmwave_4d_cloud_node`（ROS 侧增强点云）。 |

**数据流（仿真雷达）**：`gz_maritime_radar_plugin`（仿真内）→ `radar_gz_bridge` → ROS 话题 → 算法 / `usv_sim_full` / Foxglove 等。

**数据流（毫米波）**：整机主链路多为 URDF `gpu_ray` → `ros_gz_bridge` → `mmwave_4d_cloud_node` → 最终 `…/points`；独立验证世界见包内 `launch/`、`worlds/`。

---

## 目录树（逻辑结构）

```
sensor_plugins/
├── gz_maritime_radar_plugin/     # Gazebo 海事雷达插件 + 模型
├── radar_gz_bridge/              # gz → ROS2 雷达桥接
├── usv_mmwave_sim/               # 毫米波：Gazebo 插件 + ROS 点云增强节点
└── gy_radar_driver-main/         # 实机雷达（GY）驱动与 launch/config
```

---

## 与邻域的衔接

| 对象 | 关系 |
|------|------|
| [usv_sim_full](../usv_sim_full/仓库结构说明.md) | 主仿真 launch 可能拉起桥接或依赖其话题；配置常在 `usv_sim_full/config`。 |
| `src/third_party/marine_msgs` | `marine_sensor_msgs` 消息定义；`radar_gz_bridge` / `gy_radar_driver` 依赖。 |
| `usv_interfaces` | 工程统一接口与 topic 常量；驱动与桥接发布话题时宜对齐。 |
