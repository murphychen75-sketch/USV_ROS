# USV Simulation 使用指南

## 1. 项目概述

USV Simulation 是一个基于 ROS2 和 Gazebo Harmonic 的无人水面艇仿真平台。该项目基于 VRX (Volvo Robotics Competition) 框架，专为 USV (Unmanned Surface Vehicle) 系统的开发、测试和验证而设计。

## 3. 快速开始

### 3.1 环境准备

确保已正确安装 ROS2 Humble 和 Gazebo Harmonic。

### 3.2 编译项目

```bash
cd /path/to/your/workspace
colcon build --packages-select usv_sim_full
source install/setup.bash
```

### 3.3 运行仿真

```bash
# 启动主仿真
ros2 launch usv_sim_full main.launch.py config_path:=src/usv_simulation/usv_sim_full/config/full_config.yaml
```

## 4. 配置说明

### 4.1 主配置文件

位于 [usv_sim_full/config/full_config.yaml](./usv_sim_full/config/full_config.yaml)：

- **environment**: 仿真环境配置
  - `world_name`: 仿真世界名称（如 "sydney_regatta"）

- **robot**: 机器人配置
  - `name`: 机器人名称
  - `xacro_template`: 使用的 Xacro 模板
  - `package_name`: 包名称
  - `thruster_config`: 推进器配置（'H', 'T', 'X', 'CUSTOM'）
  - `spawn_pose`: 初始位姿 [x, y, z, roll, pitch, yaw]
  - `overrides`: 物理参数覆盖
    - `mass`: 质量
    - `inertia`: 惯性矩阵
    - `thruster_positions`: 推进器位置参数
  - `buoyancy_params`: 浮力和水动力参数

- **sensors**: 传感器配置
  - `lidars`: 激光雷达配置
  - `cameras`: 摄像头配置
  - `imus`: IMU配置
  - `gps_sensors`: GPS传感器配置

- **obstacles**: 障碍物配置
  - `random_areas`: 随机生成区域
  - `fixed_list`: 固定障碍物列表

- **visualization**: 可视化配置
  - `launch_rviz`: 是否启动RViz
  - `enable_telemetry`: 是否启用遥测

### 4.2 传感器配置

在配置文件中可以定义多种传感器：

```yaml
sensors:
  lidars:
    - name: "front_lidar"          # 传感器名称
      parent_link: "base_link"     # 父链接
      xyz: [0.7, 0.0, 1.8]       # 位置 [x, y, z]
      rpy: [0.0, 0.0, 0.0]       # 旋转 [roll, pitch, yaw]
      topic: "/sensors/lidar/front/points"  # 话题名称
      enabled: true                # 是否启用
```

## 5. 模型管理

### 5.1 模型目录结构

项目采用分层模型管理策略：

- **VRX 原始模型**：保持 `vrx/` 目录结构不变
- **自定义模型**：在 `usv_sim_full/models/` 中管理

### 5.2 自定义模型添加

1. 将自定义模型放置在对应的子目录中：
   - `models/assets/` - 手动维护的核心模型
   - `models/original/` - 原始设计模型
   - `models/core/` - 核心基础模型

2. 在 Xacro 文件中引用模型路径：
   ```xml
   <mesh filename="package://usv_sim_full/models/assets/my_model.stl"/>
   ```

### 5.3 传感器宏定义

在 [templates/sensor_macros.xacro](./usv_sim_full/templates/sensor_macros.xacro) 中定义传感器宏：

- `vlp16_macro`: VLP-16 激光雷达宏
- `camera_macro`: 摄像头宏
- `imu_macro`: IMU 宏
- `gps_macro`: GPS 宏

## 6. 会话管理

系统使用 [session_manager.py](./usv_sim_full/scripts/session_manager.py) 自动管理仿真会话：

1. 读取 YAML 配置文件
2. 生成参数化的 URDF 文件
3. 创建传感器桥接配置
4. 生成 RViz 配置
5. 记录会话日志到 `logs/` 目录

每次运行仿真会在 `usv_sim_full/logs/` 目录下创建带时间戳的会话文件夹，包含：
- 原始配置文件
- 参数化的 Xacro 文件
- 最终的 URDF 文件
- 传感器桥接配置
- RViz 配置文件

## 7. 启动流程

### 7.1 主启动文件

[main.launch.py](./usv_sim_full/launch/main.launch.py) 是主启动文件，它会：

1. 调用 [session_manager.py](./usv_sim_full/scripts/session_manager.py) 生成配置文件
2. 启动基础设施仿真组件
3. 启动机器人系统（状态发布、实体创建、传感器桥接）
4. 启动可视化组件

### 7.2 组件启动

- `infra_sim.launch.py`: 启动 Gazebo 环境和世界
- `robot_bringup.launch.py`: 加载机器人模型、传感器和桥接
- `visualization.launch.py`: 启动 RViz2 可视化界面

## 8. 故障排除

### 8.1 常见问题

1. **找不到模型文件**：
   - 检查 `GAZEBO_MODEL_PATH` 环境变量
   - 确认模型文件路径正确

2. **传感器未显示**：
   - 确认传感器配置中的父链接名称正确
   - 检查传感器话题是否正常发布数据

3. **物理参数异常**：
   - 检查质量、惯性参数是否合理
   - 确认浮力和水动力参数设置正确

### 8.2 调试技巧

- 查看 `logs/` 目录下的最新会话文件
- 检查生成的 URDF 文件是否包含期望的配置
- 使用 `ros2 topic list` 验证话题是否正常发布
- 使用 `rviz2` 手动添加传感器话题进行可视化验证

## 9. 开发指南

### 9.1 添加新传感器

1. 在 [templates/sensor_macros.xacro](./usv_sim_full/templates/sensor_macros.xacro) 中定义传感器宏
2. 在配置文件中添加传感器定义
3. 更新桥接配置以包含新传感器话题
4. 测试仿真以确保传感器正常工作

### 9.2 自定义船体

1. 在 [templates/](./usv_sim_full/templates/) 中创建新的 Xacro 模板
2. 在配置文件中指定使用新模板
3. 调整物理参数以匹配新船体
4. 测试仿真以确保船体行为正常
