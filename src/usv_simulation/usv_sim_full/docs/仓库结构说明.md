本文件说明 **本 ROS 2 包内部**主要目录职责。在 `usv_simulation` 中的定位与邻域关系，见总览文档（双向引用）：

**→ [../docs/docs_v3/仿真仓库结构说明.md](../docs/docs_v3/仿真仓库结构说明.md)**（章节「usv_sim_full / 核心启动包」）

更完整的功能说明、Launch 拆解与配置示例见同目录 **[README.md](./README.md)**。

---

## 目录树（主要结构）

```
usv_sim_full/                      # 包根：package.xml、setup.py、setup.cfg
├── launch/                        # 启动入口与子组件
│   ├── main.launch.py             # 全量仿真主入口（多船、毫米波/海事雷达、环境动力学等）
│   ├── test_hull.launch.py        # 船体/简化环境快速验证
│   ├── nav2_* / sensor_tune.*     # Nav2、传感器调参等相关入口
│   └── components/                # 基础设施 / 机器人 bringup / 可视化等拆分 launch
├── config/                        # YAML：整机、传感器、定位、桥接、RViz 调参等
├── description/                   # URDF/xacro、内置模型、默认 rviz 资源
├── worlds/                        # Gazebo 世界与任务场景（SDF 等）
├── rviz/                          # RViz2 配置文件
├── test_env/                      # 简化水面等测试世界
├── sh/                            # 辅助 shell 脚本（快速启动、检查等）
├── logs/                          # 会话运行产物（动态生成；勿将大文件纳入版本库）
└── usv_sim_full/                  # Python 包名与目录同名
    └── scripts/                   # 核心 Python 节点与工具
        ├── session_manager.py     # 读 YAML、生成 URDF/桥接/RViz 等会话产物
        ├── obstacle_spawner.py    # 障碍物布局与生成
        ├── usv_env_dynamics.py    # 环境/动力学相关逻辑
        ├── scenario_manager_node.py # 场景管理
        └── …                      # 遥控、TF、里程计等辅助节点
```

---

## 各目录一句话

| 路径 | 作用 |
|------|------|
| `launch/` | 组装 gz、机器人、传感器桥接、RViz；`components/` 用于模块化启动。 |
| `config/` | 仿真与机器人参数的唯一事实来源之一（多为主 YAML）。 |
| `description/` | 机器人与传感器 xacro、网格/模型路径；供 session 生成最终 URDF。 |
| `worlds/`、`test_env/` | 仿真世界与轻量测试场景。 |
| `rviz/` | 预置显示布局；运行时常与 `logs/session_*/session.rviz` 配合。 |
| `usv_sim_full/scripts/` | 运行时逻辑与 ROS 2 节点（随 `setup.py` 安装为可执行模块）。 |
| `logs/` | `session_manager` 等写入的会话目录（URDF、桥接、障碍物 JSON 等）。 |

---

## 与邻域包的关系（只读提示）

- **`sensor_plugins/`**：海事雷达 gz 插件、`radar_gz_bridge`、实机雷达驱动等；本包 launch 可能依赖其节点与话题。说明见 [../sensor_plugins/仓库结构说明.md](../sensor_plugins/仓库结构说明.md)。  
- **`vrx/*`**：WAM-V 描述与 gz 资源；本包 xacro/世界常引用 `wamv_gazebo` 等。  
- **`usv_interfaces`**：自定义话题与常量，脚本与 launch 宜与其对齐。
